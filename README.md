# BIG DATA - Caso Studio Esame (NoSQL)

## Descrizione
Questo repository raccoglie materiali di studio strutturati per la preparazione del parziale universitario su **Big Data e basi di dati NoSQL**.

Il caso studio si concentra su:
- fondamenti Big Data (3V, trade-off architetturali)
- differenze tra approccio relazionale e NoSQL
- modellazione **aggregate-oriented** e **key-value design**
- utilizzo pratico di **Redis** (tipi dati, comandi, transazioni, persistenza, indici secondari)
- simulazioni in stile prova d’esame con esercizi guidati

## Struttura del repository

### `materiali_studio_big_data/`
Dispense create a partire dai PDF del corso, organizzate per argomento:
- `00-indice-e-piano-studio.md`: percorso consigliato e obiettivi

- `02-modellazione-aggregate-key-value.md`: progettazione dati e trade-off
- `03-redis-operazioni-transazioni-indici.md`: parte operativa Redis
- `04-esercizi-guidati-e-simulazioni-parziale.md`: tracce e soluzioni in stile parziale

## Obiettivo didattico
L’obiettivo del repository è fornire una base di studio:
- **completa** (teoria + pratica)
- **concreta** (esempi reali e casi d’uso)
- **orientata all’esame** (domande tipiche, esercizi, checklist)

## Come usare il materiale
1. Parti dagli indici (`00-...`) per avere il piano di studio.
2. Studia in sequenza: Fondamenti -> Modellazione -> Redis.
3. Chiudi con le simulazioni, cercando di risolverle prima senza guardare le soluzioni.
4. Ripassa con checklist e errori frequenti prima del parziale.
